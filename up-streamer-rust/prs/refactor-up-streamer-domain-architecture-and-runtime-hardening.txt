Title: refactor: complete up-streamer domain architecture and runtime hardening

## Scope and review boundary

- This branch is 44 commits ahead of `upstream/main`.
- 10 commits are patch-equivalent to open PR #77 (issue-74) and are intentionally out of scope for this review.
- Review target for this PR is the remaining 34 commits (`e76dc3d..f16f155`).

## Why this refactor

- Split monolithic streamer orchestration into domain-owned modules so behavior is easier to reason about and change safely.
- Preserve outward `up-streamer` ergonomics while hardening runtime lifecycle paths and reducing hot-path overhead.
- Improve diagnosability with consistent structured tracing and explicit route-to-worker correlation.
- Add repeatable performance guardrails to keep future ergonomic refactors safe.

## What changed

- Domain architecture completion in `up-streamer`
  - Consolidated internal ownership into `control_plane`, `routing`, `data_plane`, and `runtime` modules.
  - Reduced orchestration density in `up-streamer/src/ustreamer.rs` and moved logic to layer owners.
  - Added layered rustdoc/doctests across core module boundaries.

- Subscription-provider decoupling and topology cleanup
  - Migrated to `USubscription`-contract refresh path with sync-health semantics.
  - Kept static mode as default and made `live_usubscription` explicit fail-fast (deferred integration).
  - Removed standalone `subscription-cache` crate by integrating cache logic into `up-streamer` routing internals.

- Route identity and API cleanup
  - Made route lifecycle errors idiomatic (`Display` + `Error`).
  - Removed forwarding-rule alias APIs and migrated call sites/docs to canonical route APIs.
  - Restored immutable URI projection keys for runtime keying and moved static dedupe to projection-key semantics.

- Ergonomics + performance guardrails
  - Added Criterion benchmark harness in `up-streamer`.
  - Added `utils/criterion-guardrail` CLI and benchmark helper script.
  - Added advisory benchmark workflow for CI signal without introducing unstable hard-fail behavior.

- Structured observability rollout
  - Added shared event/field helpers under `up-streamer/src/observability`.
  - Migrated ingress/egress/routing/control/runtime logs to structured, grepable events.
  - Added explicit `route_label` to `worker_id` correlation events.

- Runtime/perf/reliability hardening
  - Hardened plugin lifecycle ownership and ingress listener teardown symmetry.
  - Added snapshot caching and reduced routing/ingress overhead.
  - Shifted egress workers to shared runtime tasks.
  - Stabilized integration scenarios with bounded wait controls and reduced timing flakiness.

## Breaking/compatibility notes

- `UStreamer::add_forwarding_rule` and `UStreamer::delete_forwarding_rule` were removed; use `add_route` and `delete_route`.
- `live_usubscription` mode is intentionally reserved in this phase and fails fast with explicit guidance.
- Transport behavior is preserved; this PR focuses on architecture, maintainability, observability, and reliability.
